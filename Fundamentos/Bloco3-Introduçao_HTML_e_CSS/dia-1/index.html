O navegador procura o endereço IP para o nome do domínio O navegador envia uma requisição HTTP para o servidor web O servidor responde com um redirecionamento permanente ao site O browser segue o redirecionamento O servidor trata a requisição O servidor envia de volta uma resposta HTML O navegador começa a renderização da página HTML O navegador envia requisições para os objetos dentro do HTML O navegador envia mais requisições assíncronas

Luan
1. Você digita google.com na barra de endereços do seu navegador. 2. O navegador verifica se há um registro DNS no cache para encontrar o endereço IP correspondente de google.com. 3. Se o URL solicitado não estiver no cache, o servidor DNS do provedor (ISP) inicia uma consulta para encontrar o endereço IP do servidor que hospeda google.com. 4. O navegador inicia uma conexão TCP com o servidor. 5. O navegador envia uma requisição HTTP para o webserver da google. 6. O servidor lida com a requisição e envia uma resposta de volta. 7. Uma resposta HTTP é enviada pelo servidor para o cliente. 8. O navegador exibe o conteúdo, e você faz sua pesquisa.

Marcelo
Quando digitamos o endereço do google.com no campo de busca do navegador. o navegador (cliente) envia uma requisição de acesso à página, e, o servidor devolve uma resposta com a permissão de acesso, contendo todos os arquivos que formam o site.

Bianca
Tudo começa quando digitamos o endereço (URL) em um navegador web, esse navegador procura o endereço IP para o nome do domínio (DNS), o navegador envia uma requisição HTTP para o Servidor Web. Com isso, o servidor responde com um redirecionamento ao site. O Browser faz um redirecionamento tratando a requisição e envia de volta a resposta (HTML). Após isso inicia a renderização desta página web. E com isso o navegador envia requisição para o conteúdo dentro do HTML para o usuário. Concluindo com o usuário tendo visualização e acesso a essa página web.
